# AGENTS.md

> The spec - stack, architecture, direction, active work - lives in OpenSpec (`openspec/`) and ADRs. This file is standing policy only.

## Dependencies

- No copyleft dependencies or assets. Prefer Apache-2.0, MIT, BSD, or ISC.
- Track latest security-patched releases. Do not knowingly pin vulnerable versions.

## Decisions & Docs

- Record architecture-significant decisions as ADRs based on the Architecture Decision Record pattern.
- Maintain user and developer guides in Antora.
- Update docs in the same change when user- or developer-facing behavior changes; keep them short.

## OpenSpec

- Use OpenSpec for proposed product or architecture changes.
- Add an ADR task when a change affects major layers, packaging, security model, terminal architecture, runtime supervision, or dependency strategy.
- Add an Antora doc task when a change affects users or developers.
